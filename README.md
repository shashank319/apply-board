# ⚡ .NET Contract Jobs Dashboard

A zero-backend static dashboard of **US .NET contract roles**, scraped from LinkedIn via
[Apify](https://apify.com), filtered strictly, and auto-refreshed 3× per day by GitHub Actions.
Hosted free on GitHub Pages.

## What it does

- **Scrapes** job postings with the Apify Actor
  [`fantastic-jobs/advanced-linkedin-job-search-api`](https://apify.com/fantastic-jobs/advanced-linkedin-job-search-api).
- **Filters strictly** (`scripts/transform.js`) — a job is kept **only if all** are true:
  1. The literal string `.net`, `dotnet`, or `asp.net` (case-insensitive) appears in the **title or description**.
     _This drops "net zero" / "net present value" false positives that the source query lets through._
  2. Employment type is **CONTRACTOR or TEMPORARY** (no full-time / permanent / direct-hire).
  3. Country is **United States**.
- **Deduplicates** by URL and gives each job a **stable id** (`job_<hash-of-url>`) so the
  dashboard can track your "Applied" status even after the data refreshes and reorders.
- **Renders** a searchable, filterable card dashboard with applied-tracking saved in your browser.

## Files

| File | Purpose |
|------|---------|
| `index.html`, `styles.css`, `app.js` | The static dashboard (no build step, no backend). |
| `jobs.json` | The filtered job data the dashboard reads. Regenerated by the workflow. |
| `scripts/transform.js` | Turns raw Apify output into filtered `jobs.json`. Enforces the strict rules. |
| `.github/actor-input.json` | The Apify Actor search config (edit this to change the search). |
| `.github/workflows/refresh-jobs.yml` | The cron job that refreshes the data 3×/day. |

## How the data flows (automatic)

```
          ┌─────────── GitHub Actions (cron 3×/day) ───────────┐
          │                                                    │
Apify Actor ──run-sync──► raw/dataset.json ──transform.js──► jobs.json ──git commit──► GitHub Pages ──► your browser
```

1. GitHub Actions wakes on schedule (05:00 / 13:00 / 21:00 UTC).
2. It calls the Apify Actor's `run-sync-get-dataset-items` API with `.github/actor-input.json`.
3. `scripts/transform.js` filters + dedupes the results into `jobs.json`.
4. If `jobs.json` changed, it's committed and pushed.
5. GitHub Pages serves the new `jobs.json`; the dashboard fetches it on next load.

**Your "Applied" marks are never lost** — they live in your browser's `localStorage`, keyed to each
job's stable id, independent of the data file.

## One-time setup

### 1. Add your Apify token as a repository secret
- Get your token: Apify Console ▸ **Settings ▸ Integrations ▸ API token**.
- In this repo: **Settings ▸ Secrets and variables ▸ Actions ▸ New repository secret**
  - Name: `APIFY_TOKEN`
  - Value: _(paste your token)_

### 2. Enable GitHub Pages
- **Settings ▸ Pages ▸ Build and deployment ▸ Source: Deploy from a branch**
- Branch: `main`, folder: `/ (root)` ▸ **Save**.
- Your site appears at `https://<username>.github.io/<repo>/`.

### 3. (Optional) Trigger the first refresh manually
- **Actions ▸ Refresh .NET contract jobs ▸ Run workflow.**

## Changing the search or frequency

- **Search terms / location / contract types:** edit `.github/actor-input.json`.
- **How often it refreshes:** edit the `cron:` lines in `.github/workflows/refresh-jobs.yml`
  (times are UTC). Examples are in the comments — once daily, every 6 hours, etc.

## Running locally

```bash
# regenerate jobs.json from a raw Apify dump
node scripts/transform.js raw/dataset.json jobs.json

# serve the dashboard
python3 -m http.server 4173
# open http://localhost:4173
```

## Cost

The Actor is pay-per-result (~$0.005/job on the free tier + a tiny start fee). At 300 jobs × 3
runs/day that's roughly **$4–5/month**; lower Apify tiers drop the per-job price. GitHub Actions
and Pages are free for public repos.
